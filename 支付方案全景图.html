一套完整的支付方案对比、流程可视化、选型决策树与避坑指南。点击任意卡片展开详情。

微信支付

限制最多

官方明确不支持在 App WebView 中使用 H5 支付。iOS 支付完成后会跳转到外部浏览器，体验极差。App 场景应使用原生 SDK 的「APP 支付」。

H5支付 ❌ App内 JSAPI ❌ 普通浏览器 APP支付 ✅ 推荐

支付宝

相对友好

手机网站支付（Wap）在 App WebView 中可以正常工作，但需拦截 alipays:// 协议唤起支付宝 App。未安装时引导下载（新商户不再支持网页版）。

手机网站支付 ✅ 微信浏览器 ❌ APP支付 ✅ 推荐

六种支付方式全景对比

微信 H5 支付
MWEB
普通浏览器
微信内浏览器
App WebView 不推荐

适用场景：用户在普通手机浏览器（Chrome、Safari、UC）中打开商户 H5 页面，唤起微信支付。

申请条件：企业/个体户 + ICP 备案 + 微信支付商户号开通 H5 产品。

技术流程：后端统一下单（trade_type=MWEB）→ 返回 mweb_url → 前端拼接 redirect_url 跳转 → 用户在微信内确认支付 → 完成后跳回。

关键限制：支付链接有效期仅 5 分钟；referer 必须正确设置；iOS 支付后无法回跳 App。

→ 查看微信支付 H5 支付官方文档 ↗

微信 JSAPI 支付
JSAPI
普通浏览器
微信内浏览器
App WebView

适用场景：用户在微信内置浏览器（公众号 H5 页面）中完成支付。

必需条件：需获取用户 openid（需微信授权）；需配置 JS 接口安全域名；需要公众号认证。

技术流程：获取 openid → 后端统一下单（trade_type=JSAPI）→ 前端 wx.config 注入权限 → 调用 wx.chooseWXPay 或 WeixinJSBridge.invoke。

与 H5 支付的核心区别：JSAPI 必须在微信内使用，且必须传 openid。

→ 查看微信支付 JSAPI 支付官方文档 ↗

微信 Native 支付
NATIVE
普通浏览器
微信内浏览器
App WebView

适用场景：所有环境通用，用户通过扫描二维码完成支付。

技术流程：后端统一下单（trade_type=NATIVE）→ 返回 code_url → 前端生成二维码 → 用户微信扫一扫 → 完成支付。

用户体验一般：移动端需要用户切换 App 扫码，多一步操作。

→ 查看微信支付 Native 支付官方文档 ↗

微信 APP 支付
✅ 推荐
普通浏览器 不适用
微信内浏览器 不适用
App WebView

适用场景：原生 Android/iOS App 中集成微信支付。

技术流程：集成微信 SDK → 后端生成预支付订单 → 前端调用 SDK 调起微信 → 用户支付 → 返回 App → 异步通知确认。

核心优势：最佳用户体验，完整的支付结果回调，支付完成后自动返回 App。

这是微信官方对 App 场景唯一推荐的支付方式。

→ 查看微信支付 APP 支付官方文档 ↗

支付宝手机网站支付
Wap
普通浏览器
微信内浏览器
App WebView

适用场景：手机浏览器中打开商户 H5 页面，唤起支付宝完成支付。

申请条件：企业/个体户 + 实名认证 + ICP 备案。

技术流程：后端调用 alipay.trade.wap.pay → 返回 Form 表单或 URL → 前端提交/跳转 → 唤起支付宝 App → 支付完成 → 自动返回。

新商户（2019年8月后）未安装支付宝时不再支持网页支付，仅引导下载 App。

→ 查看支付宝手机网站支付官方文档 ↗

支付宝 APP 支付
✅ 推荐
普通浏览器 不适用
微信内浏览器 不适用
App WebView

适用场景：原生 Android/iOS App 中集成支付宝支付。

技术流程：集成支付宝 SDK → 后端生成订单字符串 → 前端调用 SDK 发起支付 → 用户在支付宝 App 内完成 → 返回结果。

核心优势：最佳体验，完整支付结果回调（resultCode: 9000=成功，6001=取消），支付完成后自动返回 App。

这是 App 场景下支付宝的最佳实践。

→ 查看支付宝 APP 支付官方文档 ↗

避坑指南

微信侧常见坑

原因：统一下单传的 spbill_create_ip 与用户实际调起支付时微信侧检测到的 IP 不一致。

解决：确保后端传的是用户真实 IP（不是服务器 IP）；保持网络环境一致（不要下单时 WiFi，支付时切 4G）。

→ 查看微信支付 H5 支付官方文档 ↗

原因：直接访问支付链接或 App WebView 未设置 referer。

解决：按正常页面跳转流程发起；Android WebView 中手动设置 referer：headers.put("Referer", "https://your-domain.com")。

→ 查看微信支付 H5 支付官方文档 ↗

原因：微信 H5 支付的 iOS 系统限制，支付完成后无法自动回跳 App。

解决：这是系统限制，无法根本解决。建议服务端模拟中间请求直接拿到最终支付地址；或者更推荐直接改用微信 APP 支付。

→ 查看微信支付 APP 支付官方文档 ↗

支付宝侧常见坑

原因：WebView 默认拦截了 alipays:// 等非 http 协议。

解决：重写 shouldOverrideUrlLoading，当 URL 以 alipays:// 开头时，用 Intent 唤起支付宝 App。

→ 查看支付宝手机网站支付官方文档 ↗

原因：2019年8月起新签约商户，未安装支付宝时不再支持浏览器网页支付，仅引导下载 App。

解决：这是产品策略，无法绕过。确保你的页面有"未安装支付宝"的友好提示，引导用户下载。

→ 查看支付宝手机网站支付官方文档 ↗

原因：微信会拦截支付宝的唤起链接。

解决：不要在微信浏览器中提供支付宝支付选项。检测到微信浏览器时，只展示微信支付或引导用户"在浏览器中打开"。

→ 查看支付宝手机网站支付官方文档 ↗

// Android WebView 拦截微信支付链接
@Override
public boolean shouldOverrideUrlLoading(WebView view, String url) {
    if (url.startsWith("weixin://wap/pay?")) {
        Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse(url));
        startActivity(intent);
        return true; // 拦截，不要继续加载
    }
    return super.shouldOverrideUrlLoading(view, url);
}
// Android WebView 拦截支付宝支付链接
@Override
public boolean shouldOverrideUrlLoading(WebView view, String url) {
    if (url.startsWith("alipays://") || url.startsWith("alipay")) {
        Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse(url));
        startActivity(intent);
        return true; // 拦截，唤起支付宝 App
    }
    return super.shouldOverrideUrlLoading(view, url);
}

// 前端判断浏览器类型并调起微信 H5 支付
function isWeixin() {
    var ua = window.navigator.userAgent.toLowerCase();
    return ua.match(/MicroMessenger/i) == "micromessenger";
}

核心建议

原生 App 首选原生 SDK 支付；混合开发通过桥接调用原生 SDK；纯 WebView 封装 App 中支付宝 H5 可用，微信 H5 建议评估 iOS 回跳问题。

快捷方案

预算有限或快速上线？考虑聚合支付服务商（通联支付、拉卡拉等），一套接口覆盖微信+支付宝，省去单独申请和对接的麻烦。
