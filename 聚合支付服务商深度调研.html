聚合支付服务商 是什么？

一套完整的聚合支付服务商定义、角色定位、商业模式、主要参与者、费率对比与合规风险分析。

定义

聚合支付（又称第四方支付），是相对于第三方支付而言的概念。通过聚合多种第三方支付平台、合作银行及其他支付工具，为商户提供统一接口的综合支付服务。

→ 什么是第四方支付 ↗

核心定位

不碰资金、不碰清算。聚合支付服务商不具备央行支付牌照，不能从事资金结算。仅提供支付环节的信息流转、技术对接、商户操作承载和统一对账服务。

→ 央行对聚合支付的定位 ↗

与第三方支付的区别

第三方支付（微信、支付宝、银联）提供资金清算通道，有央行支付牌照。
第四方支付提供技术集成服务，不碰资金，是"连接支付渠道的一根绳子"。

聚合支付服务商连接消费者、第三方支付平台和商户，实现"一码多付"、统一接口、统一对账

统一技术接口

一套 API 对接所有支付渠道（微信、支付宝、银联、云闪付等），无需分别对接每个平台的 SDK 和文档。大幅降低开发成本和维护复杂度。

一码多付 / 统一收银

一个收款码或一个收银页面，消费者可自行选择微信、支付宝、银行卡等任意支付方式。极大提升支付转化率，减少消费者流失。

统一对账 & 数据分析

聚合支付平台将所有渠道的交易数据统一汇总，提供统一的对账报表、交易数据分析、退款管理、分账等功能。免去商户手动汇总多个后台数据的痛苦。

合规红线：聚合支付必须知道的"四个不得"

央行明确划定聚合支付"四个不得"红线

根据《中国人民银行支付结算司关于开展违规"聚合支付"服务清理整治工作的通知》，聚合支付定位于"收单外包机构"，不得从事以下行为：

❌ 不得经手结算资金
不得以任何形式经手特约商户结算资金，从事或变相从事特约商户资金结算（即"二清"）。
❌ 不得伪造/篡改交易信息
不得伪造、篡改或隐匿交易信息。
❌ 不得存储敏感信息
不得采集、留存特约商户和消费者的敏感信息。
❌ 不得从事核心业务
不得以任何形式从事或变相从事特约商户资质审核、受理协议签订、资金结算、风险监测等收单核心业务。
→ 查看央行政策原文 ↗

什么是"二清"？为什么危险？

"二清"是指没有获得央行支付业务许可的单位或个人，在持牌收单机构的支持下实际从事支付业务和资金清算的模式。

典型流程：用户付款 → 资金先到聚合平台账户 → 平台再分发给商户。这就是二次清算。

⚠️ 风险
  • 资金沉淀在平台，平台跑路则商户资金损失
  • 涉嫌非法经营罪
  • 不受银行监管保护
  • 一旦被查，平台关停，商户资金冻结
→ 第四方支付与非法经营罪 ↗

如何识别安全的聚合支付？

资金是否官方直达？
安全的聚合支付：资金由微信/支付宝官方直接结算到商户银行卡，不经过平台中转。
平台是否有支付牌照？
持牌收单机构（通联、拉卡拉、汇付天下等）有央行支付牌照，受监管保护。纯技术服务商（Ping++）不碰资金。
到账路径是否清晰？
资金应直接进入商户自己的银行账户，而非"先进入平台再提现"。
是否有官方合作伙伴资质？
如 YunGouOS 是微信/支付宝官方授权合作伙伴，签约后商户号由微信/支付宝直接下发。

选型建议：聚合支付 vs 官方直接对接

适合选择聚合支付的场景

小微商户 / 个人开发者：没有营业执照，无法直接申请微信/支付宝商户号（如 YunGouOS、易支付支持个人开户）
需要快速上线：不想花数周申请资质、走审核流程，希望一天内接入支付
多渠道统一需求：需要同时接入微信、支付宝、银联、京东支付等，不想分别对接每个平台的 SDK 和文档
需要统一对账：多平台交易数据汇总、统一报表、自动分账、退款管理
线下门店场景：需要 POS 机、聚合收款码、扫码枪等硬件支持（通联、拉卡拉、钱方）

建议直接对接官方支付的场景

大型商户 / 高交易量：交易量大时，通过聚合支付服务商额外收取的平台服务费会显著增加成本。直接对接官方可谈更低费率
对资金安全极度敏感：需要完全掌控资金流转路径，不接受任何中间环节
有完整技术团队：有能力分别对接微信、支付宝官方 SDK，维护各平台的更新
需要深度定制化：需要利用微信/支付宝的高级营销工具（如微信分账、红包、优惠券、花呗分期等），聚合平台可能不支持或滞后
已有营业执照：企业/个体户已有资质，直接申请官方商户号的门槛并不高
对比维度 官方直接对接（微信+支付宝） 聚合支付服务商
接入复杂度 高（需分别对接两个平台 SDK、文档、审核） 低（一套 API 覆盖所有渠道）
资质要求 企业/个体户 + ICP 备案 部分平台支持个人（如 YunGouOS）
上线速度 慢（申请审核 1-4 周） 快（最快当天可用）
费率 微信 0.6% / 支付宝 0.6%（标准） 可能更低（如 0.38%）或更高（加平台服务费）
资金安全性 最高（官方直达，无中间环节） 取决于平台（选"不碰资金"的才安全）
统一对账 无（需分别登录微信/支付宝后台） 有（统一报表、统一退款）
高级功能 最全（分账、红包、营销工具等） 有限（部分平台不支持或滞后）
合规风险 最低（官方直连，受央行保护） 需注意（避免"二清"平台）
技术维护 需自行跟进各平台 API 变更 平台统一维护，SDK 自动更新
客服支持 官方客服（响应慢） 专属客服（响应快）

核心结论

聚合支付服务商是连接第三方支付与商户的桥梁，核心价值是"统一接口、一码多付、统一对账"。选择时需重点关注资金是否官方直达（避免二清），以及平台是否持有牌照或官方授权。

选型建议

小微/个人：选 YunGouOS、易支付（官方合作伙伴模式，资金安全）
企业/快速上线：选通联支付、拉卡拉（持牌机构，费率低）
技术优先：选 Ping++（纯技术封装，不碰资金）
海外/数字产品：选 LemonSqueezy、Stripe
